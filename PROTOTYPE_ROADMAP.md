# Detective Voice — 4~6주 프로토타입 로드맵

> 목적: GDD의 12개월 풀 프로덕션을 수백만 달러 태우기 전에, **혼자/소수 인원으로 게임의 가장 큰 기술 리스크 3가지를 검증**한다.

## 검증해야 할 3가지 가설

| # | 가설 | 깨지면 어떻게 되는가 |
|---|---|---|
| H1 | **사건 템플릿을 단일 진실 소스로 두면 LLM이 환각 없이 일관된 사건을 풀어낼 수 있다.** | 깨지면 게임 자체가 성립 불가. 사건이 풀리지 않음. |
| H2 | **캐릭터 카드 + 거짓말 게이트로 NPC가 압박/증거 제시에 일관되게 반응한다.** | 깨지면 심문이 무너지고, "정통 추리"가 아니라 "재미없는 챗봇" 됨. |
| H3 | **음성 입출력 지연 P50 < 800ms를 일반 모바일 환경에서 달성 가능하다.** | 깨지면 페이싱이 무너지고, "운전 중 추리극" 포지셔닝이 죽음. |

H1·H2는 음성 없이도 검증 가능 → 텍스트 프로토타입에서 먼저. H3는 그 다음.

---

## Phase 0 — 텍스트 코어 루프 (Week 1~2)

**목표**: 사건 템플릿이 진짜로 "단일 진실 소스"로 작동하는지, NPC 5명이 거짓말을 일관되게 유지하는지 CLI에서 검증.

### 산출물
- `schemas/case_template.schema.json` — 사건 템플릿 스키마
- `cases/last_customer.json` — "마지막 손님" 풀 템플릿 (NPC 5인, 단서 5종, 결말 4분기)
- `prompts/aria_gm.md` — Aria 시스템 프롬프트
- `prompts/character.md` — NPC 시스템 프롬프트 템플릿
- `detective_voice.py` — Claude Sonnet 4.6 + Haiku 4.5 호출하는 CLI 게임

### 검증 질문 (Week 1 끝)
1. AI가 사건 템플릿에 없는 사실을 만들어내는 빈도? (목표: 사건당 0~1회)
2. 같은 NPC에게 같은 질문을 3가지 다른 방식으로 했을 때 답이 일관된가?
3. 거짓말 영역 질문에 NPC가 카드의 거짓말 응답을 일관되게 사용하는가?
4. 단서를 들이밀었을 때 NPC의 불안 미터가 의도대로 변하는가?
5. Aria가 막힌 플레이어에게 "답을 주지 않으면서" 다음 행동을 제안하는가?

### 성공 기준
- 5번의 풀 플레이스루 중 4번 이상 사건이 풀린다.
- 환각 발생률 ≤ 1회/사건.
- 비-게임 작가 1명이 플레이해도 30~50분 안에 끝난다.

### 비용 가드
- 1 플레이스루 = $0.50~$1.50 (Sonnet GM + Haiku 캐릭터). 50회 돌려도 $75 미만.

## Phase 1 — 웹 음성 프로토타입 (Week 3~4)

**목표**: H3 검증. 음성 들어가도 페이싱이 살아있는지, 시각장애 사용자에게 처음 보여줄 수 있는 형태인지.

### 산출물
- `web/` — Next.js 기반 단일 페이지
  - 마이크 녹음 (브라우저 MediaRecorder)
  - STT: 1차 Web Speech API, fallback Whisper API
  - TTS: ElevenLabs 스트리밍 (Aria 1 보이스 + NPC 3~5 보이스)
  - Phase 0 코어 로직을 백엔드 API로 그대로 사용
- 지연 측정 대시보드 (P50/P95)

### 검증 질문
1. 사용자 발화 끝 → Aria 첫 음성까지 P50 지연? (목표 ≤ 800ms)
2. 시끄러운 환경(카페, 지하철) STT 정확도?
3. AirPods Pro 공간음향이 실제로 정보 채널로 작동하는가?
4. "Push to Talk" vs "Hands-free" 중 시각장애 사용자가 선호하는 모드는?

### 성공 기준
- P50 ≤ 1000ms (800ms는 stretch goal)
- 한국어 STT 정확도 ≥ 90% (조용한 환경)
- 시각장애 사용자 1~2명에게 보여주고 "이거 쓸만하다"는 직관적 반응

## Phase 2 — 클로즈드 알파 준비 (Week 5~6)

**목표**: GDD M5의 "클로즈드 알파" 직전까지 닿는 형태. 시그니처 사건 1편을 처음부터 끝까지 안정적으로 플레이 가능.

### 산출물
- "마지막 손님" 풀 시나리오에 BGM/환경음/효과음 임시 트랙
- 단서 노트 / 사건 노트 음성 호출 ("노트 읽어줘")
- 4가지 결말 분기 다 동작
- 시각장애 사용자 5명 사용성 테스트 (NPS, 사건 완료율)

### Go/No-Go 결정 포인트
Phase 2 끝에서 다음을 평가:
- **Go**: NPS 50+, 완료율 70%+ → GDD의 12개월 풀 프로덕션 착수
- **Pivot**: 일부 메커닉 재설계 필요 → 시그니처 사건 구조 또는 AI 아키텍처 수정
- **No-Go**: 풀리지 않거나 환각이 잡히지 않음 → AI 모델 수준이 더 올라올 때까지 1~2분기 대기

## 혼자/소수 인원으로 굴리는 법

| 역할 | 누구 | 시간 |
|---|---|---|
| 디렉팅 + 사건 작가 | 본인 | 풀타임 |
| 백엔드/프롬프트 엔지니어 | 본인 또는 외주 1명 (1~2주) | Phase 0 |
| 웹 프론트 + 음성 통합 | 외주 1명 (2주) | Phase 1 |
| 시각장애 사용자 베타 모집 | 한국 점자도서관 / 시각장애인복지관 협력 | Phase 2 |
| 사운드 (임시 BGM/효과음) | 라이브러리(Soundsnap/Artlist) | Phase 2 |

총 비용 추정: **$8K~$15K** (외주 1~2명 + API 비용 + 베타 사례비)

이 프로토타입이 검증되면, 그때 GDD의 $1.6M~2.5M 풀 프로덕션을 자신감 있게 착수할 수 있다.
